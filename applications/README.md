# Applications тАФ status board

Single source of truth for application status. The status of each application
lives in the `status` field of its `applications/<company-role>/job.md`; the
history lives append-only in `applications/<company-role>/events.jsonl`. **The
board, funnel and headhunter table below are GENERATED** by
[`../scripts/build_board.js`](../scripts/build_board.js) тАФ don't hand-edit them.

**Status values (hand-editable in `job.md`):** `draft` тЖТ `applied` тЖТ `screening`
тЖТ `interview` тЖТ `offer` тЖТ **terminal:** `accepted` | `rejected` | `withdrawn` ┬╖
special case `discarded` (dropped before applying).

**Event types (`events.jsonl`):** `created`, `applied`, `ack`, `screen`,
`interview`, `assessment`, `offer`, `accepted`, `rejected`, `withdrawn`,
`discarded`, `follow_up`, `note`. Rejection `reason` тИИ {`not_a_fit`, `salary`,
`location`, `process_cancelled`, `other_offer`, `no_response`, `other`}, `side` тИИ
{`company`, `me`}.

## Maintenance

**The board is generated тАФ don't hand-edit it.** To change status:

1. Edit the `status` field in `<slug>/job.md` (one edit per transition).
2. Append one line to `<slug>/events.jsonl`, e.g.
   `{"ts":"2026-08-10","type":"rejected","reason":"no_response","side":"company","note":"..."}`.
3. Run `node scripts/build_board.js` тАФ it regenerates the board, funnel and
   headhunter table between the `<!-- board:start/end -->` markers.

Notes:

- Keep folder paths stable (don't move them between status folders) so links and
  history stay intact.
- The build needs **no npm** тАФ just Node 20+ (`fs`/`path` builtins). It's
  idempotent: a second run with no changes produces no diff.
- Headhunter changes: edit `outreach/headhunters.json` (or the example file), then
  re-run the script.
